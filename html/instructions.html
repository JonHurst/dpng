Instructions

This is a proof of concept proofreading interface. It is not hooked up to the DP servers, so there are only 10 pages available; they are served in random order. You cannot do any damage, but you also cannot save your work.

It is designed to be used mainly with the keyboard. The controls are:

KeyAction
jMove both highlights down one line.
kMove both highlights up one line.
shift + jMove the image highlight down one line.
shift + kMove the image highlight up one line.
e or iEdit the text.
ESCClose the editor and re-validate.

So far, the following features have been implemented:

  • Synchronised scrolling and highlighting. A "current line" is highlighted in the image and the text to help your eye scan easily between the two versions. If the lines become un-synchronised, use the shifted versions of the movement keys to re-synchronise.
  • Resizability. Using the handle in the bottom left hand corner of the control, you can set the size. As you change the width, the image zooms in and out.
  • Continuous validation. Every time you close the editor after making a change, the text is sent to the server to be checked. So far, all punctuation is highlighted green, spelling errors are underlined amber and end of line hyphens are underlined red. The final version should also underline in green any word that is a member of a stealth scanno pair, but, since I don't yet have a list of these, for the moment you just get "he" and "be".
  • Context pages. The previous and next page images are available by scrolling the image window beyond the limits of the current page. Ultimately, this could make it simple to move hyphenated words up from the next page, thereby saving the PPers a job.

Enjoy!
